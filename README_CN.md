# SoulLoom 🧵 对话灵魂织机

> 把对话碎片织成一个可以对话的人。

SoulLoom 是一个开源框架，能将聊天对话数据转化为**带阶段感知的 AI 人格角色**。与通用的角色构建工具不同，SoulLoom 捕捉的是**一个人在不同关系阶段的表达演变**——语气习惯、幽默方式、情绪节奏、互动启发式。

---

## 为什么需要 SoulLoom？

现有的"AI 角色"方案要么：
- **手工调 prompt** — 没有数据支撑，全靠猜
- **克隆公开人物** — 千篇一律，没有个人特色
- **静态快照** — 把人物当作一成不变

SoulLoom 一次解决三个问题：**数据驱动、真实个人、阶段感知。**

## 织机隐喻

| 概念 | 含义 |
|------|------|
| **经线 (Warp)** | 稳定的核心人格 — 贯穿所有阶段不变的特质 |
| **纬线 (Weft)** | 各阶段特有的表达方式 — 随关系变化的语气、话题、幽默 |
| **梭子 (Shuttle)** | 分析过程 — 在数据中来回穿梭，把经线和纬线织在一起 |

一个人不是一成不变的。他们随以下因素变化：
- 和你的熟悉程度
- 所处的生命阶段
- 关系的具体情境

SoulLoom 捕捉的就是这个。

## 工作流程

```
用户上传聊天 JSON
    │
    ▼
Phase 1: 理线 — 数据校验、概览统计、确认目标
    ▼
Phase 2: 分色 — 时间分布分析、自动阶段识别
    │       阶段名称由内容+消息量动态生成，不套模板
    ▼
Phase 3: 定织 — 选择模式（完整/分阶段/单个/定制）
    ▼
Phase 4: 织造 — 表达DNA提取 → 人格棱镜 → 心智模型
    │       → 决策启发式 → 情绪曲线
    ▼
Phase 5: 成衣 — 生成可部署的 Claude 角色 Skill
```

## 产出物

生成 `.claude/skills/[人名]-perspective/` 目录：

```
[人名]-perspective/
├── SKILL.md                       # 完整的角色 Skill（含阶段选择器）
├── references/
│   ├── base-profile.md            # 稳定核心人格（经线）
│   ├── stages/                    # 各阶段配置（纬线）
│   └── sources/
│       └── chat_data.json         # 原始数据
```

## 使用场景

- **数字遗产** — 保存亲人的表达方式
- **创作者分身** — 把自己的聊天记录变成 AI 替你回复
- **角色创作** — 基于真实对话数据的游戏 NPC
- **自我观察** — 通过数据看自己的沟通模式
- **关系存档** — 一段重要关系的交互式记忆

## 快速开始

```bash
git clone https://github.com/Shanguier/soulloom.git
cd soulloom
# 将 SKILL.md 加载到 Claude Code 即可自动引导执行
```

**前置条件：** 聊天导出 JSON 文件（每条消息包含发送者、时间、内容）

## 验证状态

已通过真实数据验证：
- **2,445 条消息 / 7 个阶段** — 全流程测试通过
- **30,000+ 条消息 / 8 个阶段** — 规模验证通过

两个验证均生成了可运行的、带阶段感知的交互式角色。
