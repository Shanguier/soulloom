# SoulLoom · 对话灵魂织机

> 经线是人格的底色，纬线是时光的纹路。把对话碎片织成一个可以对话的人。

## 核心理念

SoulLoom 不是复制人，也不是思维框架提炼器。它做的是第三件事：**从对话数据中还原一个人的「表达人格」**。

和 nuwa-skill 的区别：

| 维度 | nuwa-skill | SoulLoom |
|------|-----------|----------|
| 数据源 | 公开信息（书籍/访谈/社交网络） | 私人对话数据（聊天记录） |
| 捕捉目标 | 思维框架（HOW they think） | 表达人格（WHO they are） |
| 产出用途 | 思维顾问 / 决策参谋 | 可对话角色 / 数字分身 |
| 表达风格 | 专业、结构化 | 自然、口语化、带关系温度 |

**织机隐喻：**

- **经线（Warp）** — 一个人稳定的核心人格特质，贯穿所有阶段不变
- **纬线（Weft）** — 每个阶段的独特表达方式，随关系和情境变化
- **梭子（Shuttle）** — 分析过程，在数据中来回穿梭，把经线和纬线织在一起

**关键区分**：输出的不是「一个像XX的 chatbot」，而是一个「在不同的关系阶段有不同表现的真实人格」—— 因为人本来就是这样的。

## 执行流程

### Phase 1: 理线（Thread Sorting）

#### Step 1: 数据接入

接收用户提供的对话数据。支持格式：

```json
[
  {"s": "发送者名称", "t": "2024-09-02 14:30", "m": "消息内容"},
  ...
]
```

必需字段：`s`（发送者）、`t`（时间）、`m`（消息内容）。

> **请把对话 JSON 文件发给我。**
>
> 格式要求：每条消息包含发送者、时间、内容。如果格式不一样，发给我看看，我来适配。

#### Step 2: 数据概览

读取后展示概览：

```
📊 数据概览
━━━━━━━━━━━━━━━━━━━━
总消息数    : 2,445 条
时间跨度    : 2024-09-02 ~ 2026-05-24（约 20 个月）
参与者      : 你 × 范士炬
消息占比    : 你 52% | 范士炬 48%
━━━━━━━━━━━━━━━━━━━━
```

> 这是要蒸馏的对象对吧？确认后进入阶段分析。

#### Step 3: 人物确认

> 你要蒸馏的是 **[对方名称]** 对吧？他自己的消息共 **N** 条，占 **X%**。

用户确认后进入 Phase 2。

---

### Phase 2: 分色（Color Separation）

#### Step 1: 时间分布分析

读取 JSON，按天/周统计消息量，识别以下特征：

| 特征 | 判断标准 |
|------|---------|
| 持续高峰 | 消息量 > 均值 × 1.5，持续 ≥ 2 周 |
| 持续低量 | 消息量 < 均值 × 0.3，持续 ≥ 2 周 |
| 断档 | 连续 ≥ 7 天零消息 |
| 爬坡 | 从低到高递增 ≥ 3 周 |
| 骤降 | 两周之间降幅 > 60% |
| 回暖 | 从低谷反弹至均值以上 |

#### Step 2: 阶段划分

基于时间分布 + 内容特征，自动切分阶段并**动态命名**——不套模板，根据实际数据说话：

**命名逻辑（两个维度取交集）：**

| 消息量特征 ↓ | 内容特征 → | 工作/事务型 | 玩乐/兴趣型 | 情感/倾诉型 | 日常/闲聊型 |
|-------------|------------|------------|------------|------------|------------|
| 爬坡期 | | 磨合期 | 试探期 | 破冰期 | 熟悉期 |
| 高峰期 | | 并肩期 | 狂欢期 | 热络期 | 热聊期 |
| 平稳期 | | 协作期 | 日常玩乐期 | 温存期 | 日常闲聊期 |
| 下降期 | | 收尾期 | 冷却期 | 疏离期 | 淡化期 |
| 断档前后 | | 重启期 | 复燃期 | 回暖期 | 恢复期 |

**实际命名由数据驱动**，上方表格只是参考坐标系。举例：

- **两个男生聊游戏开黑多、消息量高** → 不叫「热络期」，叫「开黑期」
- **两个女生聊约饭探店多** → 不叫「稳定期」，叫「约饭期」
- **工作关系转朋友** → 不叫「爬坡期」，叫「破冰期」

展示给用户：

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
共 2,445 条消息，时间跨度 2024-09-02 ~ 2026-05-24

基于消息量曲线和聊天内容，自动识别出以下阶段：

1️⃣  破冰期 —— 2024-09-02 ~ 2024-10-06（约5周，156条）
     关键词：工作协调、试探性接触、客气但保持距离

2️⃣  热络期 —— 2024-10-07 ~ 2024-11-16（约6周，477条）
     关键词：约打球、约饭、调侃、分享个人信息

     ═══ 2个月长假断档 ═══

...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ 阶段名称由聊天内容 + 消息量曲线自动生成。你觉得名字不对随时可以改。
```

用户确认阶段划分后进入 Phase 3。

### Phase 3: 定织（Pattern Design）

> **你想织成什么样？**
>
> a) **完整织锦** — 综合全部聊天记录，织成一个统一的本人画像（不分阶段）
> b) **分时织锦** — 按阶段逐个织造，生成带阶段选择器的完整 Skill
> c) **单幅小样** — 聚焦一个时期的风格
> d) **定制拼织** — 选从某阶段到某阶段，拼成一幅综合体

用户选择后执行：

- **a选** → 不切分，全部数据作为整体分析
- **b选** → 创建所有阶段配置，逐阶段并行蒸馏
- **c选** → 问用户选哪个阶段，只分析该阶段
- **d选** → 问「从哪个阶段到哪个阶段？」，合并指定范围

---

### Phase 4: 织造（Weaving）

#### 织造维度

无论哪种模式，都从以下维度逐层分析：

**1. 表达DNA**

| 维度 | 提取内容 |
|------|---------|
| 句式偏好 | 长短句比例、陈述/疑问/祈使句分布 |
| 高频词汇 | Top 高频词、专属口头禅、语气词 |
| 节奏特征 | 单条消息长度、秒回还是隔天回、断句方式 |
| 幽默方式 | 吐槽/自嘲/冷幽默/方言梗/网络梗 |
| 情绪表达 | 表情包描述、感叹号频率、情绪词汇 |

**2. 人格棱镜**

从对话中捕捉此人看人的方式、自我定位、关系认知、冲突处理模式等。每个棱镜都要有 ≥2 条聊天记录支撑。

**3. 心智模型**（3-5个）

此人在对话中反复出现的思维模式——不是「他怎么想」，是「他习惯性怎么想」：
- 务实协调者：「先确认情况→快速评估→给出方案→结伴行动」
- 平等互助者：「你帮我我帮你，不算账」
- 松弛调侃者：「正经事说完就开始嘴」

**4. 决策启发式**

他做判断时的快捷方式。格式：「如果 X，则倾向于 Y」。

**5. 情绪基调**

整体情绪底色 + 各阶段情绪变化曲线。

#### 分阶段织造

如果选 b（分时织锦），每个阶段独立分析：

```
阶段 N:
  表达DNA微调: [此阶段特有的用词/语气/节奏]
  人格棱镜活跃度: [哪些棱镜在此阶段最明显]
  决策启发式优先级: [哪些启发式在此阶段更常用]
  情绪基调: [此阶段的关系温度]
```

跨阶段**不变**的部分（经线 → `base-profile.md`）：
- 核心价值观
- 稳定的人格特质（如内向/外向、主动/被动等）
- 说话的基础风格底色

分阶段**变化**的部分（纬线 → `stages/0N-阶段名.md`）：
- 语气亲密度
- 话题偏好
- 回应速度和长度
- 幽默和吐槽的浓度

#### 输出产物

生成到 `.claude/skills/[人名]-perspective/`：

```
.claude/skills/[人名]-perspective/
├── SKILL.md                      # 完整的角色Skill（含阶段选择器 + 扮演规则）
├── references/
│   ├── base-profile.md           # 跨阶段共享人格档案（经线）
│   ├── stages/                   # 各阶段配置（纬线）
│   │   ├── 01-阶段A.md
│   │   └── ...
│   └── sources/
│       └── chat_data.json        # 原始对话数据
```

## 技术特性

- **纯本地处理**：所有分析在本地完成，数据不出机器
- **数据无侵入**：只读不写原始数据，不修改用户文件
- **格式自适应**：能适配常见聊天导出格式（JSON/CSV/TXT）
- **阶段命名动态生成**：基于内容 + 量级，不套固定模板
- **输出即插即用**：生成的是 Claude 原生 Skill，激活即用

## 依赖

- Python 3.8+
- 无第三方 API 依赖

## 诚实边界

- **仅基于文字信息**：无法捕捉语气、表情、肢体语言等非文字表达
- **仅覆盖数据覆盖的时间范围**：无法推断之前或之后的变化
- **阶段划分是人为切分**：真实人格变化是渐进的
- **聊天 ≠ 全部**：人在不同社交圈、不同场景下可能完全不同
- **关系影响表达**：对方的存在本身会影响此人的表达方式，无法完全剥离
